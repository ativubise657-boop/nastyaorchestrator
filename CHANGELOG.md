# Changelog

## v34

Quality release — без новых фич, укрепляем фундамент.

- **Устойчивость к медленным клиентам** — SSE события теперь ограничены очередью (maxsize=100), медленный/зависший клиент не съедает память сервера при длинных задачах.
- **Защита от OOM при загрузке файлов** — лимит 50 МБ на документ, 413 ошибка с понятным сообщением (раньше можно было уронить бэкенд большим файлом).
- **Чистая БД при удалении проекта** — каскадом удаляются папки, ссылки и состояние circuit breaker (раньше висели «сиротские» записи).
- **Понятные ошибки в UI** — когда backend не отвечает, теперь показывается красный тост «Backend не отвечает. Проверь что приложение запущено.» вместо пустого чата.
- **Автообновления зависимостей** — настроен Dependabot, еженедельно приходят PR-ы с minor/patch обновлениями Python/npm/Rust/GitHub Actions.

## v33

### Главное
- **Чат-сессии как в ChatGPT** — каждый чат теперь отдельная сессия со своей историей. Sidebar слева, inline-переименование (двойной клик), удаление, горячие клавиши (стрелки/Enter).
- **Изоляция картинок по сессиям** — прикреплённый в одном чате скриншот больше не виден LLM в другом. Починен баг «две картинки» когда модель путалась между `image.png` из разных сообщений.
- **Документы** — гибридный режим: PDF/TZ загруженные в проект видны во всех сессиях, clipboard-скриншоты привязаны к конкретной сессии.

### Качество кода (рев big)
- Event loop FastAPI больше не блокируется SQLite (асинхронные обёртки).
- Circuit breaker стал persistent — переживает рестарт воркера.
- Добавлен CI workflow: tsc, pytest, cargo clippy блокируют PR с регрессиями.
- 100+ новых тестов (webhooks, quality_gate, mode_resolver, gemini_executor, chat-sessions, circuit_breaker).
- Рефакторинг god-файлов: Zustand store (1127→93 строки + 9 слайсов), DocPanel (1344→644+5 подкомпонентов), ChatPanel (986→575+3), queue_next endpoint (224→40 строк + 7 extracted функций).
- ExecuteRequest dataclass для трёх LLM-executor'ов (перестали расходиться сигнатуры).
- 22 места с `except pass` теперь логируют ошибки.

### Migration
Автоматическая миграция БД при первом запуске v33: старые сообщения каждого проекта → Legacy-сессия «Chat 1», clipboard-картинки (is_scratch=1) → session-scoped, PDF-документы → project-wide.

## v32

- Починен Tauri auto-updater: связка signing-ключей, формат артефактов и генерация `latest.json` наконец дружат между собой. Кнопка «Обновить приложение» снова работает.
- В CI fail-fast, если `-setup.exe.sig` или `latest.json` не собрались — больше не будет «зелёный билд, но релиз без апдейтера».

## v0.1.3
- Проверка работы обновления с Git
- Версия приложения теперь синхронизируется между CHANGELOG, хедером и модалкой обновления.
- Проверка GitHub на Windows теперь идёт через отдельный SSH wrapper, поэтому обновление больше не должно срываться на `sh.exe`.
- GPT CLI режимы теперь явнее разделены на обычный и думающий, а ошибки авторизации и reasoning стали понятнее.

## v0.1.2

- Модалка обновления теперь показывает человеческое описание релиза из `CHANGELOG.md`, а не только список git-коммитов.
- Кнопка `Обновить приложение` живёт в хедере рядом со статусом worker и выглядит аккуратнее.
- Перед обновлением приложение показывает текущую и целевую версию, а во время установки ведёт прогресс-бар по шагам.
- Если changelog для релиза ещё не заполнен, интерфейс автоматически откатится к списку коммитов, чтобы пользователь не видел пустое окно.
